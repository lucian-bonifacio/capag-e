Primeiro
Minha ideia é substituir o bloco atual de “Após homologação” por uma
  subseção mais completa, não adicionar mais uma seção redundante.

  Hoje temos:

  Após homologação:

  - se o usuário aprovar...
  - se o usuário pedir ajuste...
  - não marque TASK...

  E logo depois:

  Se `scope-resolution` classificar...

  Eu substituiria esses dois blocos por uma única subseção dentro de ##
  5. Fluxo De Trabalho, algo como:

  ### Continuidade Após Execução

  Após informar o usuário e aguardar homologação, feche o estado atual
  antes de seguir.

  - Se o usuário aprovar a TASK, use `execution-log` para registrar
  aprovação, use `roadmap-manager` para marcar `concluido` e recalcular a
  `## Próxima Tarefa`.
  - Depois de concluir uma TASK, informe a nova `## Próxima Tarefa` e
  peça autorização simples antes de executá-la.
  - Se o usuário pedir ajuste dentro da TASK atual, use `scope-
  resolution`, mantenha ou retorne a TASK para `pendente` e peça
  autorização simples antes de executar o ajuste.
  - Se `scope-resolution` classificar a solicitação como nova TASK, peça
  confirmação explícita. Após aprovação, use `task-planner` e `roadmap-
  manager` para criar e posicionar a TASK. Não implemente a nova TASK no
  mesmo passo.
  - Se a solicitação exigir nova SPEC, decisão governada ou remoção de
  bloqueio, pare e aguarde decisão do usuário.
  - Não marque TASK como `concluido` sem homologação do usuário.
  - Não avance automaticamente para outra TASK após concluir, ajustar ou
  criar tarefa.

  Isso cobre:

  - homologação aprovada;
  - ajuste;
  - nova TASK;
  - nova SPEC;
  - bloqueio;
  - próxima tarefa;
  - proibição de avanço automático.

  Fica mais limpo do que manter “Após homologação” e criar outra seção
  depois.
