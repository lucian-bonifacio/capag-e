# 06 - Roadmap de execucao do Modulo 03

## Objetivo

Ordenar as tarefas do Modulo 03 em sequencia cronologica.

## Sequencia proposta

1. Revisar a `SPEC-06`.
2. Confirmar mapeamento entre nomes atuais e nomes canonicos.
3. Definir entidade de dominio `CapagEAssessment`.
4. Definir metodos permitidos.
5. Definir status permitidos.
6. Definir regras de bloqueio.
7. Adaptar contrato de API planejado.
8. Adaptar linguagem de frontend planejada.
9. Adaptar exportacao planejada.
10. Planejar testes por camada.
11. Revisar compatibilidade com Modulo 01.
12. Revisar dependencia com Modulos 04, 05, 06 e 07.

## Entregavel esperado

Ao final do modulo, o projeto deve ter uma especificacao pronta para implementar:

- contrato canonico;
- status;
- metodos;
- bloqueios;
- payload;
- exportacao;
- testes.

## Criterios de aceite

- A `SPEC-06` esta completamente coberta.
- O sistema passa a distinguir `FCO` de `FCA`.
- `PLR ajustado` passa a ter equivalencia formal com `PLRA`.
- `CAPAG-E` nao pode ser apresentada sem metodo e status.
