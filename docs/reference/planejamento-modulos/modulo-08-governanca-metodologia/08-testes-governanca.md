# 08 - Testes de governanca

## Testes obrigatorios

1. Validar colunas obrigatorias dos assets PLR/FCO atuais.
2. Validar que todo componente referenciado existe.
3. Validar tratamentos permitidos.
4. Validar que laudo/exportacao inclui versao metodologica.
5. Validar que manual/spec de estado atual nao contradiz motores implementados em pontos canonicos.

## Areas afetadas

- `assets`
- `docs`
- `tests`
- `export`
- `report`

## Criterio de sucesso

Divergencia entre metodologia documentada e comportamento real deve ser detectavel.
