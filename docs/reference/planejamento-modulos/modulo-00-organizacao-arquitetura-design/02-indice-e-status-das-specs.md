## Detalhamento

- Classificar cada spec como:
  - implementada;
  - parcialmente implementada;
  - futura;
  - governanca;
  - defasada.
- Definir espaco para specs novas:
  - camada declarada;
  - plano referencial oficial e metodologia;
  - frontend rotas;
  - design system;
  - modulo comportamental.
- Relacionar specs com codigo e tasks.

## Entregavel

Proposta de indice oficial de specs.

## Criterios de sucesso

- Fica claro o papel de cada spec.
- Fica claro quais specs novas precisam ser criadas.
